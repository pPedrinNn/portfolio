Projetos Realizados

CRUD 1-N com .NET e PostgreSQL

Tecnologias: ASP.NET Core MVC, PostgreSQL, Entity Framework Core

Projeto prático de desenvolvimento de um sistema de cadastro com relacionamento 1 para N entre fornecedores e produtos.

Passos Principais:

  1. Criação do projeto MVC no Visual Studio
  2. Configuração do PostgreSQL e Entity Framework
  3. Definição das classes Model (Fornecedor e Produto)
  4. Criação do DbContext e configuração da conexão
  5. Geração de migrations e criação do banco
  6. Scaffolding de controllers e views
  7. Personalização da interface

Exemplo de Configuração:

// Configuração do DbContext no Program.cs
builder.Services.AddDbContext<ApplicationDbContext>(options =>
    options.UseNpgsql(builder.Configuration.GetConnectionString("DefaultConnection")));

Modelo de Fornecedor:

public class Fornecedor
{
    public int FornecedorId { get; set; }
    public string Nome { get; set; } = string.Empty;
    public string Cidade { get; set; } = string.Empty;
    public ICollection<Produto>? Produtos { get; set; }
}

Aprendizados

Esta disciplina me proporcionou um entendimento profundo sobre programação orientada a objetos e sua aplicação em projetos reais. Aprendi a importância de conceitos como encapsulamento, herança e polimorfismo para criar sistemas mais organizados e manuteníveis.

O projeto prático de CRUD com .NET e PostgreSQL foi essencial para consolidar o conhecimento sobre arquitetura MVC, Entity Framework e relacionamentos entre entidades. A implementação do relacionamento 1-N entre fornecedores e produtos me mostrou na prática como modelar corretamente as classes e configurar o DbContext para refletir esses relacionamentos no banco de dados.

Além disso, aprendi sobre padrões de projeto e como eles podem ser aplicados para resolver problemas comuns de desenvolvimento de software de forma elegante e eficiente. A arquitetura em camadas e o padrão MVC se mostraram particularmente úteis para organizar o código e separar adequadamente as responsabilidades.
